# JLX130-026-PN 液晶模块 ESP32-C3 驱动开发手册
## 一、模块基础信息
| 项目         | 参数/说明                          |
|--------------|------------------------------------|
| 驱动IC       | ST7789V                            |
| 显示分辨率   | 240×240 点阵                       |
| 供电电压     | 3.3V（禁止接5V，避免烧毁）|
| 通信方式     | 4线SPI（仅需RS、SCK、SDA三个GPIO） |
| 显示色彩     | 16位RGB彩色（65K色）|
| 视角         | IPS全视角                          |
| 背光         | 2颗LED（直接接3.3V常亮）|

## 二、硬件接线（仅3个GPIO）
### 2.1 接线表（ESP32-C3 → JLX130-026-PN）
| JLX130-026-PN 引脚 | 功能         | ESP32-C3 连接方式                | 备注                     |
|--------------------|--------------|----------------------------------|--------------------------|
| 1-4（NC）| 空脚         | 悬空                             | 无需接线                 |
| 5（LDEA）| 背光正极     | ESP32-C3 3.3V                    | 背光常亮，无需GPIO控制   |
| 6（VSS）| 电源地       | ESP32-C3 GND                     | 必须共地                 |
| 7（VDD）| 电源正极     | ESP32-C3 3.3V                    | 核心供电                 |
| 8（SCK）| SPI时钟      | ESP32-C3 GPIO（例：GPIO4）| 自定义，需和代码一致     |
| 9（SDA）| SPI数据      | ESP32-C3 GPIO（例：GPIO5）| 自定义，需和代码一致     |
| 10（RS/A0）| 指令/数据选择 | ESP32-C3 GPIO（例：GPIO6）| 自定义，需和代码一致     |
| 11（RST）| 复位         | ESP32-C3 3.3V（通过10K上拉电阻） | 硬件固定复位，节省GPIO   |
| 12（CS）| 片选         | ESP32-C3 GND                     | 永久选中，节省GPIO       |

### 2.2 接线注意事项
1. 所有接线必须使用3.3V电平，ESP32-C3的GPIO不支持5V，切勿接错；
2. 建议在VDD和VSS之间并联一个100nF电容，稳定供电；
3. 接线长度尽量短（≤20cm），避免SPI信号干扰导致显示乱码；
4. RST接3.3V后，模块上电自动复位，无需软件控制；若需软复位，可改为接GPIO（需占用第4个GPIO）。

## 三、开发环境准备
### 3.1 基础环境
- ESP-IDF版本：v5.0及以上（兼容ESP32-C3）；
- 开发工具：VSCode + ESP-IDF插件（推荐）；
- 编译目标：`esp32c3`。

### 3.2 核心依赖
无需额外安装库，仅需使用ESP-IDF自带的：
- `driver/gpio.h`：GPIO控制；
- `freertos/FreeRTOS.h`：延时函数；
- `esp_log.h`：日志输出（可选）。

## 四、驱动代码实现
### 4.1 完整驱动代码
```c
#include <stdio.h>
#include "driver/gpio.h"
#include "freertos/FreeRTOS.h"
#include "freertos/task.h"
#include "esp_log.h"

// ===================== 配置区（根据实际接线修改）=====================
#define LCD_RS_GPIO    GPIO_NUM_6  // RS/A0引脚
#define LCD_SCK_GPIO   GPIO_NUM_4  // SCK引脚
#define LCD_SDA_GPIO   GPIO_NUM_5  // SDA引脚
#define LCD_TAG        "JLX130-026-PN"
// ===================================================================

// 颜色定义（16位RGB565）
#define COLOR_BLACK    0x0000
#define COLOR_RED      0xF800
#define COLOR_GREEN    0x07E0
#define COLOR_BLUE     0x001F
#define COLOR_WHITE    0xFFFF

// GPIO初始化
static void lcd_gpio_init(void)
{
    // 配置RS、SCK、SDA为输出模式
    gpio_config_t io_conf = {
        .pin_bit_mask = (1ULL << LCD_RS_GPIO) | (1ULL << LCD_SCK_GPIO) | (1ULL << LCD_SDA_GPIO),
        .mode = GPIO_MODE_OUTPUT,
        .pull_up_en = GPIO_PULLUP_ENABLE,  // 上拉增强稳定性
        .pull_down_en = GPIO_PULLDOWN_DISABLE,
        .intr_type = GPIO_INTR_DISABLE
    };
    gpio_config(&io_conf);

    // 初始电平
    gpio_set_level(LCD_SCK_GPIO, 0);
    gpio_set_level(LCD_SDA_GPIO, 0);
    gpio_set_level(LCD_RS_GPIO, 0);
}

// 软件SPI写8位数据
static void lcd_spi_write_byte(uint8_t data)
{
    uint8_t i;
    for(i = 0; i < 8; i++)
    {
        // 拉低时钟，准备发送数据
        gpio_set_level(LCD_SCK_GPIO, 0);
        vTaskDelay(1 / portTICK_PERIOD_MS);  // 延时保证时序（可根据实际调整）
        
        // 发送最高位
        if(data & 0x80)
        {
            gpio_set_level(LCD_SDA_GPIO, 1);
        }
        else
        {
            gpio_set_level(LCD_SDA_GPIO, 0);
        }
        
        // 拉高时钟，让模块采样数据
        gpio_set_level(LCD_SCK_GPIO, 1);
        vTaskDelay(1 / portTICK_PERIOD_MS);
        
        data <<= 1;  // 左移，准备下一位
    }
    // 时钟恢复低电平
    gpio_set_level(LCD_SCK_GPIO, 0);
}

// 写指令
static void lcd_write_cmd(uint8_t cmd)
{
    gpio_set_level(LCD_RS_GPIO, 0);  // RS=0：写指令
    lcd_spi_write_byte(cmd);
}

// 写数据
static void lcd_write_data(uint8_t data)
{
    gpio_set_level(LCD_RS_GPIO, 1);  // RS=1：写数据
    lcd_spi_write_byte(data);
}

// 写16位数据（适配16位色值）
static void lcd_write_data_16(uint16_t data)
{
    lcd_write_data((data >> 8) & 0xFF);  // 高8位
    lcd_write_data(data & 0xFF);        // 低8位
}

// 设置显示窗口（x_start,y_start → x_end,y_end）
static void lcd_set_window(uint16_t x_start, uint16_t y_start, uint16_t x_end, uint16_t y_end)
{
    // 设置列地址
    lcd_write_cmd(0x2A);  // CASET指令
    lcd_write_data_16(x_start);
    lcd_write_data_16(x_end);
    
    // 设置行地址
    lcd_write_cmd(0x2B);  // RASET指令
    lcd_write_data_16(y_start);
    lcd_write_data_16(y_end);
    
    // 开始写显存
    lcd_write_cmd(0x2C);  // RAMWR指令
}

// 模块初始化
static void lcd_init(void)
{
    lcd_gpio_init();
    
    // 上电延时（替代硬件复位，RST接3.3V自动复位）
    vTaskDelay(200 / portTICK_PERIOD_MS);

    // ST7789V初始化指令序列
    lcd_write_cmd(0x11);  // 退出休眠模式
    vTaskDelay(120 / portTICK_PERIOD_MS);  // 等待休眠退出
    
    lcd_write_cmd(0x36);  // 设置显示方向（0x00：默认方向，可改0x60旋转90°）
    lcd_write_data(0x00);
    
    lcd_write_cmd(0x3A);  // 设置像素格式为16位
    lcd_write_data(0x05);
    
    lcd_write_cmd(0x21);  // 开启显示反转（可选，根据需求调整）
    
    lcd_write_cmd(0x29);  // 开启显示
    ESP_LOGI(LCD_TAG, "JLX130-026-PN初始化完成");
}

// 填充指定区域颜色
void lcd_fill_color(uint16_t x_start, uint16_t y_start, uint16_t x_end, uint16_t y_end, uint16_t color)
{
    uint32_t i;
    uint32_t pixel_num = (x_end - x_start + 1) * (y_end - y_start + 1);
    
    lcd_set_window(x_start, y_start, x_end, y_end);
    for(i = 0; i < pixel_num; i++)
    {
        lcd_write_data_16(color);
    }
}

// 测试任务：全屏填充不同颜色
void lcd_test_task(void *pvParameters)
{
    lcd_init();
    
    while(1)
    {
        // 全屏红色
        lcd_fill_color(0, 0, 239, 239, COLOR_RED);
        vTaskDelay(2000 / portTICK_PERIOD_MS);
        
        // 全屏绿色
        lcd_fill_color(0, 0, 239, 239, COLOR_GREEN);
        vTaskDelay(2000 / portTICK_PERIOD_MS);
        
        // 全屏蓝色
        lcd_fill_color(0, 0, 239, 239, COLOR_BLUE);
        vTaskDelay(2000 / portTICK_PERIOD_MS);
        
        // 全屏白色
        lcd_fill_color(0, 0, 239, 239, COLOR_WHITE);
        vTaskDelay(2000 / portTICK_PERIOD_MS);
        
        // 全屏黑色
        lcd_fill_color(0, 0, 239, 239, COLOR_BLACK);
        vTaskDelay(2000 / portTICK_PERIOD_MS);
    }
}

void app_main(void)
{
    // 创建LCD测试任务（优先级1，栈大小4096）
    xTaskCreate(lcd_test_task, "lcd_test_task", 4096, NULL, 1, NULL);
}
```

### 4.2 代码说明
1. **配置区**：修改`LCD_RS_GPIO`、`LCD_SCK_GPIO`、`LCD_SDA_GPIO`为实际接线的GPIO号；
2. **软件SPI**：通过GPIO模拟SPI时序，无需占用ESP32-C3的硬件SPI外设，仅用3个通用GPIO；
3. **核心函数**：
   - `lcd_set_window`：设置显示区域，是所有显示操作的基础；
   - `lcd_fill_color`：填充指定区域颜色，可用于画点、画矩形、全屏填充；
   - `lcd_test_task`：测试任务，循环显示不同颜色，验证模块是否正常工作。

## 五、编译与烧录
### 5.1 项目配置
1. 在VSCode中创建ESP-IDF新项目，将上述代码替换`main.c`；
2. 按`Ctrl+Shift+P`，执行`ESP-IDF: Configure ESP-IDF Extension`：
   - Target Chip：选择`esp32c3`；
   - Serial Port：选择开发板对应的串口；
   - Flash Size：选择`4MB`（ESP32-C3常见配置）；
   - Baud Rate：推荐`921600`。

### 5.2 烧录步骤
1. 按住开发板`BOOT`键，按下`RESET`键，松开`RESET`后再松开`BOOT`，进入下载模式；
2. 执行`ESP-IDF: Flash your project`，等待烧录完成；
3. 烧录完成后，开发板自动复位，模块开始循环显示红、绿、蓝、白、黑五种颜色。

## 六、进阶功能扩展（可选）
### 6.1 显示字符/汉字
1. 准备字库（如8×16英文字库、16×16汉字库），存储为数组；
2. 基于`lcd_set_window`和`lcd_write_data_16`，逐点绘制字符/汉字像素。

### 6.2 显示图片
1. 将图片转换为240×240的RGB565格式数组（可使用在线图片转数组工具）；
2. 通过`lcd_set_window`设置全屏，循环写入图片数组数据。

### 6.3 调整显示方向
修改`lcd_init`中`0x36`指令的参数：
- 0x00：默认方向；
- 0x60：顺时针旋转90°；
- 0xC0：顺时针旋转180°；
- 0xA0：顺时针旋转270°。

## 七、常见问题排查
| 问题现象               | 排查方向                                   |
|------------------------|--------------------------------------------|
| 模块无显示、背光不亮   | 检查VDD/VSS供电是否正常，LDEA是否接3.3V    |
| 显示乱码、颜色错误     | SPI时序延时过短（增加`vTaskDelay`时长）、接线松动 |
| 烧录成功但程序不运行   | 确认Target Chip为`esp32c3`，Flash Size配置正确 |
| 部分区域显示异常       | `lcd_set_window`的坐标是否超出240×240范围   |

## 八、总结
1. JLX130-026-PN仅需RS、SCK、SDA三个GPIO即可驱动，CS/RST通过硬件接法固定，节省ESP32-C3资源；
2. 核心是通过软件模拟SPI时序，无需依赖硬件SPI，适配性更强；
3. 基础驱动仅需实现“写指令/数据”“设置显示窗口”“填充颜色”三个核心功能，进阶功能可基于此扩展；
4. 烧录前务必确认ESP32-C3进入下载模式，且硬件接线与代码配置一致。
