硬件总体架构

芯片/主控
- ESP32-C3，低功耗、WiFi/BLE 双模，具备良好的 IFMDA/DMA 能力，适合本地 UI + 网络通信任务。

工作电源
- 3.3V 供电（部分模块如 MAX98357 功放需要 5V），带稳压与降噪，确保显示、音频、传感器等模块在低功耗模式下稳定运行。

显示屏模块
- 型号：ST7789 240x240 LCD
- 通信接口：4线 SPI 接口
- 引脚分配：
  - GPIO6: SCL（SPI 时钟）
  - GPIO7: SDA（SPI 数据 MOSI）
  - GPIO10: CS（片选）
  - 额外需要：DC（数据/命令选择）、RST（复位）、BL（背光）引脚
- 驱动：支持 DMA 优化的 SPI 传输

麦克风模块
- 型号：INMP441 I2S 数字麦克风
- 接口：I2S 接口
- 引脚分配：
  - GPIO0: LRC/WS（I2S 字选择）
  - GPIO1: BCLK（I2S 时钟）
  - GPIO2: SD（数据输入）
- 注意：INMP441 的 L/R 引脚接地选择左声道

功放模块
- 型号：MAX98357 I2S 音频功放
- 接口：I2S 接口
- 引脚分配：
  - GPIO0: LRCK（I2S 左右声道时钟）
  - GPIO1: BCLK（I2S 时钟）
  - GPIO3: DIN（数据输出）
- 供电：5V（可获得更大输出功率，典型 3.2W@4Ω）
- 注意：I2S 输入引脚支持 5V 容忍，可直接接收 ESP32-C3 的 3.3V 信号

引脚复用说明
- GPIO0/GPIO1 被麦克风和功放共用，通过软件切换 I2S 输入/输出模式
- GPIO2（麦克风数据）与 GPIO3（功放数据）分开，避免冲突

EC11 旋钮编码器
- 引脚分配：
  - GPIO4: A相
  - GPIO5: B相
  - GPIO21: 按键
- 功能：实现界面前后滚动，以及按键回退/进入功能
- 电气：需实现去抖、脉冲计数逻辑

独立设置键
- 引脚：GPIO9
- 功能：当前界面的回退/进入设置二级菜单切换，短按触发回退

温湿度传感器（可选）
- 通过网络天气接口获得

无线网络
- WiFi 功能：ESP32-C3 内置 WiFi，支持作为 STA 模式，连接至局域网并实现时间同步和天气查询

存储与持久化
- NVS（非易失性存储）用于保存用户设置、历史计数、时区/地区、最近连接的 WiFi 信息等

设计要点
- 低功耗策略：在无活动时自动进入深睡或休眠模式，保留必要的 RTC、WiFi 状态
- EMI/电磁兼容性：对显示、无线、音频信号的线缆走向与地线设计进行分布，降低噪声
- 模块化：驱动层与应用层分离，便于未来替换显示屏、麦克风、声学件等

电气注意事项
- 显示屏（GPIO6/7/10）作为 SPI 信号线对外输出，确保供电稳压和信号整洁
- 音频模块（GPIO0/1/2/3）采用 I2S 接口，GPIO0/1 共用时钟，GPIO2/3 分别用于输入输出
- EC11 编码器对抖动敏感，应在固件中实现抖动滤波和去抖策略
- 设置键应设定硬件消抖 + 短时/长时按键识别
- 5V 和 3.3V 电源独立供电，避免相互干扰；必须共地

硬件扩展与测试要点
- 在 PCB 设计阶段，尽量留出备用脚位
- 初次上电，应进行自检流程，确认显示屏、编码器、WiFi 模块均工作正常
- 软件上应支持对各外设的错误处理，确保断网/异常时设备仍能保持 UI 的基本显示与本地计时功能
